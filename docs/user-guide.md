# 用户使用手册

`11408 错题复盘助手` 是一个给个人使用的手机端学习工具。它帮助你把做错的 11408 题拍照保存下来，记录卡点，安排复习，并在报告里看到自己最容易出错的知识点。

## 第一次登录

1. 打开部署后的网址。
2. 进入“登录”页。
3. 输入邮箱和密码。
4. 如果还没有账号，切换到“注册”。
5. 登录成功后会回到首页或刚才想访问的页面。

如果页面提示 Supabase 未配置，说明当前部署还没有连接后台，不能保存真实数据。

## 添加到手机桌面

iPhone Safari：

1. 打开网站。
2. 点击底部分享按钮。
3. 选择“添加到主屏幕”。
4. 确认名称为 `11408 错题复盘助手`。

Android Chrome：

1. 打开网站。
2. 点击右上角菜单。
3. 选择“安装应用”或“添加到主屏幕”。
4. 安装后从桌面图标打开。

## 拍题上传

1. 进入“拍题”。
2. 点击上传区域。
3. 可以直接拍照，也可以从相册选择题图。
4. 确认预览清晰。
5. 如果图太大，系统会先尝试压缩；压缩失败会自动使用原图。
6. 图片最大建议不超过 8MB。

## 选择科目

保存前选择题目所属科目：

- 数学
- 数据结构
- 计算机组成原理
- 操作系统
- 计算机网络

科目会影响错题库筛选、报告统计和冲刺模式排序。

## 选择掌握状态

掌握状态表示这道题当时错在哪里：

- 完全没思路：看到题后不知道从哪里开始。
- 有一点思路：知道可能考什么，但步骤很模糊。
- 思路对但卡住：方向正确，中间某一步不会继续。
- 计算错误：方法会，但算错、抄错或细节错。
- 做对但不稳：这次做对了，但没有把握下次还能做对。
- 完全掌握：已经能稳定做对。

不要为了好看而选“完全掌握”。这个状态会影响后续复习频率。

## 写一句卡点备注

备注不用很长，只写最关键的一句，例如：

```text
递归边界没想清楚
```

好的备注应该说明你当时卡在哪里，而不是只写“不会”。

## 保存错题

填好科目、掌握状态和备注后，点击保存。

成功后系统会：

- 把原题图片保存到 Supabase Storage。
- 在错题表里生成一条记录。
- 生成初始复习计划。
- 生成一份 mock 分析结果，后续可以再点 AI 分析。

如果图片上传成功但数据库写入失败，系统会尝试清理刚上传的图片，避免留下孤立文件。

## 查看错题详情

进入“错题”，点击某一道题。

详情页可以查看：

- 原题图片。
- 题目文字。
- 科目、章节、知识点。
- 掌握状态。
- 错因标签。
- 正确思路摘要。
- 一句话提醒。

如果图片太小，可以点击图片放大查看。

## 点击 AI 分析

在错题详情页点击“分析这道题”或“重新分析”。

AI 分析会尝试补充：

- 题目文字。
- 章节。
- 知识点。
- 错因标签。
- 正确思路摘要。
- 一句话提醒。
- 复习优先级。

如果没有配置 OpenAI API Key，系统会使用 mock fallback，页面不会因为缺少 AI 配置而崩溃。

## 核对题目文字

AI 识别的题目文字可能不准。你可以在详情页点击“编辑错题”，修改题干并设置核对状态：

- `ai_unverified`：AI 识别过，但还没人工核对。
- `verified`：你已经人工核对，题干可信。
- `needs_fix`：题干明显有问题，之后要修。

当题干是 `verified` 时，重新分析默认不会覆盖题目文字。只有你明确确认允许覆盖，才会替换。

## 完成今日复习

进入“复习”页。

页面会显示今天到期和已经逾期的任务。每道题可以选择：

- 仍不会。
- 有进步。
- 已掌握。
- 复习后又错。

点击后系统会写入复习结果，并按规则调整后续复习计划。按钮写入中会禁用，避免重复点击造成重复记录。

## 查看报告

进入“报告”页，可以切换：

- 日报。
- 周报。
- 月报。

报告由后台定时任务写入。如果暂时没有报告，页面会显示空状态，不会白屏。

报告里会展示：

- 新增错题。
- 已分析错题。
- 完成复习。
- 逾期复习。
- 已掌握。
- 复习后又错。
- 科目分布。
- 高频错因。
- 薄弱知识点。
- 下一步建议。

## 使用考前冲刺模式

进入“冲刺”页。

冲刺模式会优先显示更危险的错题，例如：

- 复习后又错。
- 仍不会。
- 完全没思路。
- 已逾期。
- 需要人工核对。
- 高危知识点。

你可以按科目筛选，点击“查看详情”，或在确认掌握后点击“已掌握”。

## 导出数据

进入“设置”页。

支持导出：

- JSON：适合备份。
- Markdown：适合自己阅读和整理。
- CSV：适合用 Excel 查看。

导出只包含当前登录用户自己的数据。图片不会被打包，只会保留 Storage 路径。

## 常见问题

**页面提示 Supabase 未配置怎么办？**

说明当前部署缺少后台环境变量。需要开发者检查 `NEXT_PUBLIC_SUPABASE_URL` 和 `NEXT_PUBLIC_SUPABASE_ANON_KEY`。

**上传图片失败怎么办？**

确认已经登录，图片是 jpg、jpeg、png 或 webp，大小不超过限制。仍失败时让开发者检查 Storage bucket 和 policy。

**AI 分析失败会影响错题保存吗？**

不会。错题保存和 AI 分析是分开的。没有 OpenAI Key 时会走 mock fallback。

**报告为空正常吗？**

刚开始使用时正常。需要先上传错题、完成复习，并等待后台报告任务运行，或让开发者手动触发报告函数。

**可以删除错题吗？**

当前版本支持基础批量管理，但不做批量删除。删除类操作要谨慎，后续阶段再评估。
